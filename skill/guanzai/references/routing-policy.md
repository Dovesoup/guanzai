# Routing policy

## Stable functions

1. Orchestrator: decomposition, budget, dependency order, synthesis.
2. Intelligence analyst: research, finance, citations, evidence quality.
3. Product and systems architect: scope, UX, architecture, acceptance criteria.
4. Builder: implementation, tests, artifacts, operational execution.
5. Auditor: independent verification, security, financial and factual risk.

These are functions, not mandatory headcount. Combine functions for small tasks and instantiate parallel workers only when independence helps.

## Escalation

Choose the cheapest available model that can meet the required quality. Escalate one step after a failed verification, unresolved disagreement, low-confidence evidence, or critical irreversible impact. Premium high/xhigh reasoning is justified for architecture, funds, security, compliance, difficult code, and final adjudication. Fast mode remains forbidden at every tier.

The measured standalone Codex CLI worker overhead is about 20k input tokens in the current desktop environment. Keep short tasks inline. Use a slim task packet without conversation history when a worker is justified. WorkBuddy Hy3 used 6,619 input and 1,893 output tokens for a bounded rule review at zero credits, making it suitable for reviews, classification, research drafts, and batch analysis when its result is independently verifiable.

Audit is based on action and impact. A read-only security summary is not automatically audited; a one-line billing-formula change is. Decision-grade valuation, investment risk, and cash-flow conclusions remain independently reviewed even when read-only.

## Provider registry

Provider entries declare capabilities rather than implied authority: `plan`, `execute`, `select_model`, `select_reasoning`, context limits, relative cost, availability, and policy status. New models enter through registry updates and evaluation; role prompts remain stable.

Current preference: WorkBuddy Hunyuan 3 while free, then DeepSeek V4 Pro. Both default to high reasoning regardless of task simplicity; conserve Codex quota, not these domestic-model credits. GLM is blocked by default. Codex premium models handle the smallest critical slice.

## Memory promotion

Store private user preferences outside the public seed. Project memory may contain decisions and validated pitfalls. Promote a lesson only after evidence or repeated successful use; include scope, provenance, confidence, and review date. Retire stale lessons instead of accumulating them forever.
