# GuanZai vs. Superpowers v6.1.1

Date: 2026-07-11

## Conclusion

Continue GuanZai, but do not rebuild Superpowers' software-development methodology. Treat Superpowers as a development execution engine beneath GuanZai's general routing, provider, cost, risk, finance, and memory governance.

## What v6.1.1 actually changed

The v6.1.1 release itself mainly fixes Codex hook auto-discovery, removes orphaned Codex hook code, corrects plugin metadata, and adds deterministic Codex portal packaging. It is not primarily a new router release.

However, the v6.1.1 `subagent-driven-development` skill now contains substantial cost-aware routing guidance: cheap models for mechanical tasks, standard models for integration, strongest models for architecture and final review, explicit model selection for every subagent, and the warning that extra turns can erase a cheaper model's nominal savings. It also uses fresh context, slim task briefs, file handoffs, and a durable progress ledger.

## Overlap

- Role-based subagent delegation.
- Complexity-aware model choice.
- Fresh, bounded worker context.
- Independent review and escalation.
- Parallelism only for independent work.
- Durable task progress to survive compaction.

## Differences

| Dimension | Superpowers v6.1.1 | GuanZai v0.2.2 |
|---|---|---|
| Primary scope | Software development methodology | General work orchestration and governance |
| Entry condition | Approved development design/plan | Any task, including solo/no-spawn decisions |
| Providers | Harness-native subagents | Codex plus WorkBuddy, extensible registry |
| Economics | Qualitative model-tier rules and context hygiene | Measured startup threshold and orchestration-value gate |
| Safety | Code review and workflow quality | Action-impact audit override, finance/security/irreversibility |
| Finance | No dedicated evidence/decision mode | Stable intelligence analyst and mandatory decision-grade review |
| Memory | Development progress ledger | Planned private, project, role, and public-seed memory layers |
| Evolution | Skills and process refinement | Provider registry, evaluation, memory promotion/retirement |

## Strategic boundary

Reuse Superpowers for brainstorming, implementation plans, TDD, debugging, subagent-driven development, code review, and branch completion. GuanZai should decide whether those workflows are worth invoking, which provider/model tier handles each bounded task, whether independent audit is mandatory, and what validated lessons survive.
