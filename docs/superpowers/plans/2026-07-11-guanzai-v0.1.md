# GuanZai v0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use subagent-driven-development (recommended) or executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build an installable, cost-aware multi-agent planning skill that initializes per-project policy and emits an executable routing manifest without claiming unavailable model-switching authority.

**Architecture:** A dependency-free Python core reads a versioned TOML model registry and converts a task into role-scoped work items. A CLI exposes `init`, `plan`, and `doctor`; a compact Codex Skill tells future tasks when to invoke it and how to honor host capability limits. Provider adapters report whether they can plan, select a model, select reasoning effort, or execute.

**Tech Stack:** Python 3.11+ standard library, `unittest`, TOML, Markdown/YAML Skill metadata.
